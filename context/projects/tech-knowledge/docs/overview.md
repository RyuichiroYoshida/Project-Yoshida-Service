# 技術知識管理システム概要

## 目的

このプロジェクトの目的は、技術記事を集めるだけではなく、読んだ内容、学んだこと、再利用できる知識を構造化して保存することである。

長期的には、読書履歴と技術学習を自分の資産にし、AI エージェント、Codex CLI、Codex Cloud、将来のツールから再利用できるようにする。

## 背景

現在は Chrome Discover などで技術記事の要約を見る機会があるが、全文を読み切ることは少ない。必要なのは単なる RSS リーダーや要約ツールではなく、以下を支援する仕組みである。

- 技術トレンドを短時間で把握する。
- 読む価値のある記事を判断する。
- 有用な記事を保存する。
- 記事を要約する。
- 自分の学びを追記する。
- 後から検索・再利用できる知識に変換する。
- AI エージェントへ構造化されたコンテキストを渡す。

## コアコンセプト

このシステムでは、コンテキストを 2 種類に分ける。

- 共有コンテキスト: 複数プロジェクトで再利用できる知識、ルール、経験、実践知。
- プロジェクト固有コンテキスト: 技術知識管理システムだけに適用する具体的な仕様や判断。

## 重要な問い

このシステムは「何の記事を読んだか」だけではなく、次の問いに答えられる必要がある。

- 何を学んだか。
- なぜ有用だったか。
- 自分の開発作業とどう関係するか。
- AI が後から再利用できるか。
