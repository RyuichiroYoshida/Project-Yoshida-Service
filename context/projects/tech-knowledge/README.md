# 技術知識管理プロジェクトコンテキスト

このディレクトリは、技術知識管理システムに固有の文書、ルール、プロンプト、ワークフロー、設計判断を管理する。

## 目的

技術記事を単なる URL や要約として保存するのではなく、自分が何を学び、なぜ有用で、どう再利用できるかを構造化して残す。

## 配下ディレクトリ

- `context/`: このプロジェクト固有の文脈、ルール、知識、実践知。
- `docs/`: プロジェクト文書。
- `prompts/`: AI エージェント向けプロンプト。
- `agents/`: エージェント定義。
- `workflows/`: 作業手順。
- `decisions/`: 設計判断。
