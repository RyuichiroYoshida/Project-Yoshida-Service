# コンテキストリンクルール

## 目的

記事、知識カード、経験、ルール、設計判断のつながりを後から追えるようにする。

## ルール

- `related` には原則としてリポジトリ内の相対パスを書く。
- 外部記事の URL は本文の `参照元` または記事メタデータに書く。
- 記事から抽出した知識カードには、元記事へのリンクを書く。
- 知識カード同士は、関連概念として相互にリンクしてよい。
- 古い文書を置き換える場合は、新旧両方にリンクを書く。

## リンクを書く場面

- この記事からこの知識カードを作った。
- この経験からこのルールを追加した。
- この ADR がこの要件を決めた。
- このプロンプトがこのワークフローで使われる。

## 避けること

- 関係が弱いファイルを大量に `related` へ入れる。
- 存在しないファイルを確定情報のようにリンクする。
- URL だけを保存し、何を学んだかを書かない。
